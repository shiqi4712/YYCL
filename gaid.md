# 少儿编程体验课教师 — 用户异议处理模拟沟通练习环境

## 1. 项目概述
构建一个基于 Web 的模拟沟通训练平台，面向线上少儿编程机构的**体验课教师 / 课程顾问**。通过 AI 驱动的虚拟家长角色，模拟体验课前后常见的家长异议场景，帮助教师反复练习沟通话术、提升转化率，并为管理层提供可量化的培训数据。

## 2. 目标用户
| 角色 | 核心诉求 |
|------|---------|
| 体验课教师 / 课程顾问 | 在低风险环境中反复练习应对家长拒绝、比价、犹豫等场景，积累实战经验 |
| 培训主管 / 销售经理 | 快速复制顶尖顾问的沟通技巧，统一团队话术标准，降低新人培训成本 |
| 机构管理层 | 获得团队沟通能力的数据化评估，识别薄弱环节 |

## 3. 核心功能模块

### 3.1 角色扮演对话引擎（核心）
- **AI 虚拟家长**：基于大模型扮演多种类型的家长（价格敏感型、效果怀疑型、时间冲突型、决策权不在型等）
- **多轮对话**：支持开放式自由对话，AI 家长根据教师的回复动态调整情绪和决策倾向
- **情绪模拟**：家长在对话中表现出犹豫、抗拒、质疑、松动等情绪变化
- **成交/流失判定**：对话结束时，根据沟通质量判定"家长"是否愿意报名正价课

### 3.2 异议场景库
| 场景分类 | 典型异议 |
|---------|---------|
| 价格异议 | "太贵了"、"别人家更便宜"、"先不交钱" |
| 效果异议 | "孩子能学会吗"、"学编程有什么用"、"和别家有什么区别" |
| 时间异议 | "孩子作业多没时间"、"先试试再看" |
| 决策异议 | "要和孩子爸爸商量"、"孩子不感兴趣"、"再考虑考虑" |
| 信任异议 | "没听过你们机构"、"老师资质怎么样" |

- 支持按场景单练、随机组合、专项强化三种训练模式

### 3.3 实战回放与点评
- **对话回放**：完整保存练习记录，支持教师自主复盘
- **AI 智能点评**：从话术结构、情绪安抚、价值传递、促单技巧等维度自动评分并给出改进建议
- **金牌话术推荐**：针对教师表现不佳的环节，推送优秀话术范例

### 3.4 团队管理与数据看板
- **练习量统计**：每人每周练习次数、时长、覆盖场景数
- **能力雷达图**：价格应对、效果说服、信任建立、促单能力等多维度能力评估
- **排行榜与激励**：团队内 PK，激发练习积极性
- **主管点评**：管理层可对具体对话记录进行人工批注

## 4. 交互流程设计

```
教师登录
  → 选择训练模式（场景单练 / 随机实战 / 专项考核）
    → 阅读家长画像与背景信息（孩子年龄、上过体验课、已知顾虑点）
      → 进入对话界面（教师打字 / 语音输入，AI 家长回复）
        → 对话结束（家长成交 / 流失 / 教师主动结束）
          → 查看 AI 评分报告与改进建议
            → 保存至个人成长档案
```

## 5. 技术栈建议（Web 端）

| 层级 | 推荐方案 | 理由 |
|------|---------|------|
| 前端 | Vue3 + TypeScript + Element Plus | 轻量，国内生态成熟，上手快 |
| 后端 | **Node.js (NestJS 或 Express)** | 复用现有基础，渐进升级，前后端统一语言 |
| AI 引擎 | **DeepSeek** | 国内稳定，成本低，合规无跨境风险 |
| 数据库 | **SQLite** | 先跑起来，按需升级，零运维负担 |
| 缓存 | Node-cache（内存） | 避免 Redis 运维负担，单机场景足够 |
| 部署 | **阿里云轻量服务器** | 性价比高，国内访问快，零额外成本 |

## 6. MVP 范围（最小可行产品）
1. 支持 5-8 个核心异议场景的单练模式
2. AI 家长基础对话能力（文本交互）
3. 对话结束后的基础评分（1-5 星 + 简短文字建议）
4. 教师个人历史记录列表
5. 管理员后台查看团队成员练习数据

## 7. 后续迭代方向
- **语音对话**：接入 ASR/TTS，模拟真实电话沟通场景
- **视频面试模拟**：模拟视频通话中的表情与肢体语言训练
- **真实案例库**：接入机构历史成单/丢单录音，提炼真实对抗场景
- **AI 教练陪练**：实时对话中 AI 悄悄提示"现在可以抛优惠了"或"先处理情绪再讲逻辑"
- **与 CRM 打通**：教师实战数据与真实业绩关联分析
