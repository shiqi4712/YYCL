## 1. 项目概述

本文档为《少儿编程体验课教师异议处理模拟训练系统》的工程实现框架，面向**多 Agent 并行开发**设计。通过清晰的模块边界、统一的接口契约和独立的数据 ownership，确保不同开发单元可并行推进、独立测试、最终联调集成。

### 1.1 技术栈锁定

| 层级 | 技术方案 | 版本建议 |
|------|---------|---------|
| 前端 | Vue 3 + TypeScript + Element Plus + Vite | Vue 3.4+, TS 5.x |
| 后端 API | Node.js + Express + TypeScript | Node 20 LTS |
| ORM | Prisma | 5.x |
| 数据库 | SQLite | 3.x |
| 缓存 | node-cache | 5.x |
| AI 引擎 | DeepSeek API | v1 |
| 语音识别 | 浏览器 Web Speech API（前端 ASR） | — |

## 2. 多 Agent 并行分工

### 2.1 分工总览

| Agent | 代号 | 负责范围 | 产出物 |
|-------|------|---------|--------|
| **Agent-A** | `frontend-teacher` | 教师端前端（登录、场景大厅、对话页、点评页、个人中心） | `packages/frontend-teacher/` |
| **Agent-B** | `frontend-admin` | 培训师后台前端（账号管理、场景管理、话术库、数据看板） | `packages/frontend-admin/` |
| **Agent-C** | `backend-core` | 后端核心服务（认证、账号、场景/话术 CRUD、文件服务） | `packages/backend-core/` |
| **Agent-D** | `backend-ai` | AI 对话引擎服务（对话状态机、DeepSeek 接入、AI 点评生成） | `packages/backend-ai/` |
| **Agent-E** | `backend-data` | 数据统计服务（教师成长数据、团队汇总报告、图表数据聚合） | `packages/backend-data/` |

### 2.2 并行原则

1. **接口先行**：Agent-C/D/E 先共同定义并输出 `api-contract.yaml`（OpenAPI 3.0），Agent-A/B 基于契约开发，可并行 Mock
2. **数据库统一**：Agent-C 负责 Prisma Schema 唯一源文件，所有 Agent 共用
3. **独立可运行**：每个 Agent 的代码包独立 `npm run dev` 可启动，通过环境变量指向本地/远程依赖服务
4. **统一规范**：共享 `packages/shared/` 包（DTO 类型、工具函数、常量枚举）

## 4. 数据库设计（Prisma Schema）

> **Owner**: Agent-C（backend-core）维护 `prisma/schema.prisma`，其他 Agent 只读使用。

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

// ==================== 用户与账号 ====================
model User {
  id           Int      @id @default(autoincrement())
  username     String   @unique
  passwordHash String   @map("password_hash")
  role         UserRole // TRAINER | TEACHER
  realName     String?  @map("real_name")
  status       UserStatus @default(ACTIVE) // ACTIVE | DISABLED
  createdAt    DateTime @default(now()) @map("created_at")
  updatedAt    DateTime @updatedAt @map("updated_at")

  // 关系
  createdTeachers Teacher[] @relation("TrainerToTeacher")
  conversations   Conversation[]
  favoriteScripts TeacherFavorite[]

  @@map("users")
}

model Teacher {
  id        Int      @id @default(autoincrement())
  userId    Int      @unique @map("user_id")
  user      User     @relation(fields: [userId], references: [id])
  createdBy Int      @map("created_by") // 创建该教师的培训师 User.id
  trainer   User     @relation("TrainerToTeacher", fields: [createdBy], references: [id])
  createdAt DateTime @default(now()) @map("created_at")

  @@map("teachers")
}

// ==================== 场景与话术 ====================
model Scenario {
  id                Int              @id @default(autoincrement())
  title             String
  category          ObjectionCategory
  parentProfile     String           @map("parent_profile")
  initialMessage    String           @map("initial_message")
  difficulty        Difficulty
  status            ContentStatus    @default(ACTIVE)
  recommendedScripts String?         @map("recommended_scripts") // JSON [1,2,3]
  createdBy         Int              @map("created_by")
  createdAt         DateTime         @default(now()) @map("created_at")
  updatedAt         DateTime         @updatedAt @map("updated_at")

  conversations Conversation[]

  @@map("scenarios")
}

model Script {
  id         Int              @id @default(autoincrement())
  category   ObjectionCategory
  title      String
  content    String
  keyPoints  String           @map("key_points")
  status     ContentStatus    @default(ACTIVE)
  createdBy  Int              @map("created_by")
  createdAt  DateTime         @default(now()) @map("created_at")
  updatedAt  DateTime         @updatedAt @map("updated_at")

  favoritedBy TeacherFavorite[]

  @@map("scripts")
}

model TeacherFavorite {
  id        Int      @id @default(autoincrement())
  teacherId Int      @map("teacher_id")
  teacher   User     @relation(fields: [teacherId], references: [id])
  scriptId  Int      @map("script_id")
  script    Script   @relation(fields: [scriptId], references: [id])
  createdAt DateTime @default(now()) @map("created_at")

  @@unique([teacherId, scriptId])
  @@map("teacher_favorites")
}

// ==================== 对话与消息 ====================
model Conversation {
  id         Int                @id @default(autoincrement())
  teacherId  Int                @map("teacher_id")
  teacher    User               @relation(fields: [teacherId], references: [id])
  scenarioId Int                @map("scenario_id")
  scenario   Scenario           @relation(fields: [scenarioId], references: [id])
  status     ConversationStatus @default(ACTIVE)
  finalScore Int?               @map("final_score") // 0-100
  aiReview   String?            @map("ai_review") // JSON 存储点评详情
  startedAt  DateTime           @default(now()) @map("started_at")
  endedAt    DateTime?          @map("ended_at")

  messages Message[]

  @@map("conversations")
}

model Message {
  id             Int      @id @default(autoincrement())
  conversationId Int      @map("conversation_id")
  conversation   Conversation @relation(fields: [conversationId], references: [id])
  role           MessageRole // TEACHER | AI
  content        String
  messageType    MessageType @default(TEXT) @map("message_type")
  createdAt      DateTime @default(now()) @map("created_at")

  @@map("messages")
}

// ==================== 枚举 ====================
enum UserRole {
  TRAINER
  TEACHER
}

enum UserStatus {
  ACTIVE
  DISABLED
}

enum ObjectionCategory {
  PRICE
  EFFECT
  TIME
  DECISION
  TRUST
}

enum Difficulty {
  EASY
  MEDIUM
  HARD
}

enum ContentStatus {
  ACTIVE
  INACTIVE
}

enum ConversationStatus {
  ACTIVE
  COMPLETED
  FAILED
  TIMEOUT
}

enum MessageRole {
  TEACHER
  AI
}

enum MessageType {
  TEXT
  VOICE
}
```

## 5. 接口契约设计

> **Owner**: Agent-C/D/E 联合定义，输出 `docs/api-contract.yaml`。以下为关键接口摘要。

### 5.1 认证与账号（Agent-C）

```
POST   /api/auth/login          # 登录，返回 JWT
POST   /api/auth/logout         # 登出
GET    /api/auth/me             # 获取当前登录用户信息

POST   /api/users/teachers      # 培训师创建教师账号（仅 TRAINER）
GET    /api/users/teachers      # 培训师查看教师列表（分页、搜索）
PUT    /api/users/teachers/:id  # 修改教师状态（启用/禁用）
```

### 5.2 场景与话术（Agent-C）

```
# 场景管理（培训师）
POST   /api/scenarios
GET    /api/scenarios
GET    /api/scenarios/:id
PUT    /api/scenarios/:id
DELETE /api/scenarios/:id
PATCH  /api/scenarios/:id/status

# 话术库（培训师 + 教师）
POST   /api/scripts
GET    /api/scripts
GET    /api/scripts/:id
PUT    /api/scripts/:id
DELETE /api/scripts/:id
POST   /api/scripts/:id/favorite    # 教师收藏
DELETE /api/scripts/:id/favorite    # 取消收藏
GET    /api/scripts/favorites        # 教师查看收藏列表
```

### 5.3 AI 对话引擎（Agent-D）

```
POST   /api/chat/sessions           # 创建对话 session（传入 scenario_id）
POST   /api/chat/:sessionId/message # 教师发送消息（文本或语音转文本后的文本）
GET    /api/chat/:sessionId         # 获取对话完整消息流
POST   /api/chat/:sessionId/end     # 结束对话（主动结束，触发 AI 点评）

POST   /api/review/generate         # 为指定对话生成 AI 点评（异步或同步）
GET    /api/review/:conversationId  # 获取已生成的点评
PUT    /api/review/:id/feedback     # 教师对点评标记有帮助/不准确
```

**对话消息交互时序**：

```
教师端                        backend-ai                      DeepSeek API
  |                              |                                  |
  |-- POST /api/chat/sessions -->|                                  |
  |<-- { sessionId, initialMsg }--|                                 |
  |                              |                                  |
  |-- POST /api/chat/:id/message ->|                                |
  |                              |-- 构建 Prompt（含场景设定+历史）--->|
  |                              |<----------- AI 回复 ------------|
  |<-- { role: "ai", content } ---|                                |
  |                              |                                  |
  |-- POST /api/chat/:id/end ---->|                                |
  |                              |-- 构建 Review Prompt ---------->|
  |                              |<-------- 点评 JSON ------------|
  |<-- { review, score } ---------|                                |
```

### 5.4 数据统计（Agent-E）

```
GET    /api/stats/teacher/overview       # 教师个人概览（累计次数、平均评分等）
GET    /api/stats/teacher/trend          # 教师评分趋势（近30天折线数据）
GET    /api/stats/teacher/radar          # 教师能力雷达图数据
GET    /api/stats/teacher/history        # 教师练习记录列表

GET    /api/stats/team/overview          # 培训师查看团队总览
GET    /api/stats/team/heatmap           # 场景分类练习热度
GET    /api/stats/team/ranking           # 团队能力短板排行
GET    /api/stats/team/report            # 阶段性汇总报告（支持日期范围）
GET    /api/stats/team/export            # 导出 Excel/PDF
```

## 6. AI 引擎接入设计（Agent-D）

### 6.1 DeepSeek API 接入

```typescript
// services/deepseek.service.ts
interface DeepSeekRequest {
  model: 'deepseek-chat';
  messages: { role: 'system' | 'user' | 'assistant'; content: string }[];
  temperature: number;
  max_tokens: number;
}

class DeepSeekService {
  async chat(messages: Message[], scenario: Scenario): Promise<string>;
  async generateReview(messages: Message[], scenario: Scenario): Promise<ReviewResult>;
}
```

### 6.2 Prompt 工程规范

| Prompt 文件 | 职责 | 关键变量 |
|------------|------|---------|
| `parent-role.prompt.ts` | 定义 AI 虚拟家长的角色卡 | 场景分类、家长画像、性格特征、情绪状态 |
| `review.prompt.ts` | 定义对话点评的评判标准 | 完整对话记录、点评维度、输出 JSON Schema |

**点评输出 JSON Schema**：
```json
{
  "overallScore": 78,
  "overallStar": 4,
  "dimensions": {
    "emotionComfort": { "score": 75, "comment": "..." },
    "valueDelivery": { "score": 82, "comment": "..." },
    "objectionHandling": { "score": 70, "comment": "..." },
    "closingAbility": { "score": 65, "comment": "..." },
    "compliance": { "score": 95, "comment": "..." }
  },
  "suggestions": ["建议1", "建议2"],
  "recommendedScripts": [1, 3]
}
```

### 6.3 对话状态机

```
CREATED -> ACTIVE -> [COMPLETED | FAILED | TIMEOUT]
```

- `CREATED`：session 创建，未开始对话
- `ACTIVE`：教师已发送第一条消息，对话进行中
- `COMPLETED`：AI 家长表达成交意向或教师主动结束且评分合格
- `FAILED`：AI 家长明确拒绝且不再继续
- `TIMEOUT`：轮次达到上限（20轮）自动结束

## 9. 开发规范 checklist

- [ ] 所有 API 返回统一包装格式：`{ code: number, data?: T, message?: string }`
- [ ] 密码使用 bcrypt 哈希，禁止明文存储
- [ ] 所有 Trainer 权限接口必须通过 `requireRole('TRAINER')` 中间件
- [ ] AI Prompt 修改后需记录版本，便于 A/B 测试与回滚
- [ ] 前端语音输入仅作为发送方式，后端只接收文本，不存储音频文件
- [ ] 数据库敏感字段（passwordHash）不在任何 API 响应中暴露
- [ ] 对话消息按 `createdAt` 正序排列，确保回放一致性
